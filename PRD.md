# 高校社团招新智能匹配平台 — PRD 设计稿 V1.0

## 1. 产品概述

### 1.1 产品定位

面向高校社团招新场景的 AI 智能匹配平台（微信小程序），以同济大学“百团大战”为首发试点，通过 AI 匹配推荐、结构化信息呈现和线上线下联动，帮助新生高效发现并加入适合自己的社团，同时帮助社团精准触达目标新生。

### 1.2 目标用户

| 角色 | 画像描述 | 核心诉求 |
| --- | --- | --- |
| 新生 | 大一/研一新生，对校园社团了解有限，时间精力有限 | 快速找到匹配自身兴趣、时间、预算、性格的社团 |
| 社团负责人/干事 | 社团管理层，负责招新策划与执行 | 精准触达目标人群，高效收集和管理报名信息 |
| 学校管理方（社团联/团委） | 社团管理部门，负责审批、监督 | 规范招新流程，掌握社团活跃度数据，保障合规 |

### 1.3 核心价值

1. 匹配效率提升：AI 基于新生画像与社团画像进行双向智能匹配，将“大海捞针”变为“精准推荐”。
2. 信息透明化：结构化呈现费用、时间、校区、零基础友好度等关键决策信息，消除信息不对称。
3. 线上线下联动：打通招新前“线上了解” → 招新中“线下高效逛” → 招新后“线上入社”的完整链路。

## 2. 用户痛点与需求分析

### 2.1 新生视角

| 痛点编号 | 痛点描述 | 典型场景 |
| --- | --- | --- |
| N1 | 信息分散不透明：社团介绍散落在公众号、QQ 群、传单中，费用/时间等关键信息不清楚 | 想了解舞蹈社学费，搜遍公众号找不到明确价格 |
| N2 | 兴趣匹配难：100+ 社团不知从何选起，缺乏筛选工具 | “我喜欢音乐但不会乐器，有适合的社团吗？” |
| N3 | 时间空间限制：百团大战仅中午 1.5h，嘉定校区学生参与受限 | 嘉定新生赶到四平路，只剩半小时可逛 |
| N4 | 社恐不友好：现场嘈杂拥挤，内向型学生主动攒人社交成本高 | i 人不敢上前问“零基础能加入吗？” |
| N5 | 现场找摊困难：百团大战地图无搜索功能，找特定社团效率低 | 拿着纸质地图找了 20 分钟没找到目标摊位 |
| N6 | 重复填写信息：每个社团单独收集报名信息，格式不一 | 一下午填了 5 份不同格式的报名表 |

### 2.2 社团视角

| 痛点编号 | 痛点描述 | 典型场景 |
| --- | --- | --- |
| C1 | 触达目标人群难：信息推广依赖线下传单和朋友圈，覆盖面有限 | 小众社团在百团大战摊位前门可罗雀 |
| C2 | 信息传达不精准：现场噪音大，无法深入介绍，新生走马观花 | 精心准备的社团特色被一句“扫码了解”带过 |
| C3 | 报名管理混乱：纸质表单录入费时，微信群管理无序 | 200 份报名表手动录入 Excel 到凌晨 |
| C4 | 信息更新不及时：招新时间、地点变更无法及时通知已报名新生 | 临时换教室面试，微信群通知漏发多人 |
| C5 | 推广渠道分散：需同时维护公众号/小红书/QQ 空间等多平台 | 同一份招新文案复制粘贴到 5 个平台 |

### 2.3 学校管理方视角

| 痛点编号 | 痛点描述 |
| --- | --- |
| S1 | 社团活跃度难量化：缺乏统一的数据口径来评估社团招新和运营质量 |
| S2 | 合规审查成本高：部分社团收费不规范、虚假宣传难以监管 |
| S3 | 活动效果难评估：百团大战投入大量场地和人力，但缺少转化数据 |

### 2.4 补充痛点（调研推导）

| 编号 | 痛点 | 来源 |
| --- | --- | --- |
| E1 | 信息时效性差：社团活动时间/地点在学期中常变动，旧信息误导新生 | 竞品分析 |
| E2 | 线下排队拥挤：热门社团摊位排长队，冷门社团无人问津 | 百团大战现场观察 |
| E3 | 后续跟进断裂：招新结束后新生与社团失联，实际入社率低 | 社团管理痛点 |

## 3. 核心场景与用户旅程

### 3.1 场景 A：招新前——新生线上发现与筛选社团

用户：大一新生小张，刚入学，对社团了解为零，性格偏内向

目标：在百团大战前锁定 3-5 个感兴趣的社团

流程：

扫码/搜索小程序
→ 快速画像采集（兴趣方向 / 空闲时间 / 预算区间 / 校区 / 性格倾向，约 30 秒）
→ AI 推荐社团列表（按匹配度排序，标注“零基础友好”“嘉定可参与”等标签）
→ 查看社团详情卡片（费用结构、活动频率、成员规模、往期活动照片）
→ 收藏 / 一键预报名 / AI 问答（“零基础能跟上吗？”）
→ 生成“百团大战逛摊清单”（含摊位位置 + 建议路线）

缓解痛点：N1（信息透明）、N2（匹配难）、N4（社恐友好，先线上了解再线下接触）、N5（路线规划）

### 3.2 场景 B：招新当天——线下高效逛摊 + 线上辅助

用户：同一个小张，百团大战当天中午仅 1.5h 时间

目标：在有限时间内高效完成目标社团的了解和报名

流程：

打开小程序 → “我的逛摊清单”（收藏的社团 + 摊位地图标注）
→ 地图导航到第一个目标摊位
→ 扫摊位二维码 → 弹出社团卡片 + “一键报名”
→ 逛完目标摊位后 → AI 推荐“你路过的摊位中还有这些可能感兴趣”
→ 未逛完的收藏社团 → 线上完成报名
→ 活动结束 → 推送“你已报名的社团后续安排”

缓解痛点：N3（时间限制）、N5（找摊难）、N6（重复填表，统一报名信息）、E2（排队，线上报名分流）

## 5. 信息结构与数据要素

### 5.1 社团侧字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 社团名称 | 文本 | ✅ | — |
| 类型标签 | 多选 | ✅ | 如：文艺/体育/学术/公益/科技/社会实践 |
| 一句话简介 | 文本 | ✅ | 50 字内 |
| 详细介绍 | 富文本 | ❌ | 支持图片、视频 |
| 适合人群 | 多选 | ✅ | 如：零基础友好/需一定基础/面向特定专业 |
| 活动频率 | 单选 | ✅ | 如：每周 1-2 次/每月 2-3 次/不定期 |
| 常规活动时间 | 时间段 | ✅ | 如：周六下午 |
| 费用结构 | 结构化表单 | ✅ | 分项填写：会费/器材费/课时费/活动费 + 各项金额或范围 |
| 所在校区 | 多选 | ✅ | 四平路/嘉定/沪西/沪北 |
| 线下摊位位置 | 地图标注 | ❌ | 百团大战期间填写 |
| 往期活动照片/视频 | 媒体 | ❌ | 最多 9 张图 + 1 个视频 |
| 联系方式 | 文本 | ✅ | 微信群/公众号/负责人联系方式 |
| 成员规模 | 数字 | ❌ | 当前活跃成员数 |

### 5.2 新生侧字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 年级 | 单选 | ✅ | 大一/研一/其他 |
| 学院/专业 | 文本 | ❌ | 用于专业相关社团推荐 |
| 兴趣方向 | 多选 | ✅ | 预设标签 + 自定义输入 |
| 空闲时间段 | 多选 | ✅ | 如：工作日晚上/周末上午 |
| 预算区间 | 单选 | ❌ | 如：0/100 以内/100-300/300+ |
| 性格倾向 | 单选 | ❌ | 偏内向/偏外向/都可以 |
| 所在校区 | 单选 | ✅ | — |
| 是否有相关基础 | 多选 | ❌ | 如：乐器/舞蹈/编程/运动 |

### 5.3 AI 匹配逻辑（产品视角）

AI 根据以下维度进行双向匹配排序：

1. 兴趣匹配（权重最高）：新生兴趣标签 ↔ 社团类型标签的重合度
2. 时间兼容：新生空闲时段 ↔ 社团常规活动时间的重叠度
3. 校区可达：同校区优先，跨校区标注通勤成本
4. 预算适配：新生预算区间 ↔ 社团年度费用总额
5. 性格适配：内向型新生优先推荐“零基础友好”“氛围轻松”标签的社团
6. 基础匹配：有相关基础的新生可推荐进阶型社团

## 6. AI 能力使用设想

### 6.1 智能匹配与推荐

- 做什么：基于新生画像与社团画像计算匹配度并排序推荐。
- 用户感知：“一进来就给我推了 5 个特别对口的社团，不用翻 100 个列表了。”

### 6.2 自动生成内容

- 社团摘要版介绍：从详细介绍中提取关键信息，生成 50 字精简版卡片文案。
- i 人友好版文案：自动生成更温和、细致的介绍版本，主动回答“零基础行吗？”“社恐去了会尴尬吗？”等疑虑。
- 费用一句话说明：从结构化费用数据中生成简明说明。
- 推广文案生成：自动生成适配公众号长文、小红书图文、朋友圈短文案的招新推广内容。

### 6.3 智能问答

- 做什么：新生在社团详情页或全局搜索框中，以自然语言提问，AI 基于该社团的结构化信息和介绍内容即时回答。
- 典型问题：“摄影社需要自带相机吗？”“辩论队零基础能加入吗？训练频率高吗？”
- 用户感知：不用当面社交就能获得想知道的信息，降低心理门槛。

### 6.4 智能路线规划

- 做什么：根据新生收藏的社团清单和摊位位置，自动生成最优逛摊路线。
- 用户感知：“按推荐路线走，1 小时逛完了 8 个目标摊位，没走冤枉路。”

## 7. 成功指标与验证方式

### 7.1 核心指标

| 指标 | 定义 | 目标（首版） |
| --- | --- | --- |
| 新生匹配有效率 | 新生点击/收藏 AI 推荐社团的比例 | ≥ 40% |
| 报名转化率 | 浏览社团详情 → 完成报名的转化率 | ≥ 15% |
| 社团信息完整度 | 填写全部必填字段 + ≥2 个选填字段的比例 | ≥ 70% |
| 问答使用率 | 使用 AI 问答功能的新生占比 | ≥ 20% |
| 满意度（NPS） | 新生和社团负责人的净推荐值 | ≥ 30 |

### 7.2 辅助指标

- 新生：人均收藏/报名社团数、平均停留时长、路线规划使用率
- 社团：报名人数、信息查看量（PV/UV）、通知送达率
- 平台：DAU/MAU（招新季）、AI 问答准确率（人工抽检）

### 7.3 验证方式

| 阶段 | 方式 | 说明 |
| --- | --- | --- |
| 上线前 | 小范围内测（50 名新生 + 10 个社团） | 验证核心流程跑通、匹配推荐质量、信息结构合理性 |
| 上线后 1 周 | 埋点数据分析 + 用户问卷 | 追踪关键转化漏斗，收集定性反馈 |
| 百团大战当天 | 现场 AB 对照观察 | 对比使用平台的新生 vs 未使用的新生在逛摊效率和报名数上的差异 |
| 招新季结束 | 深度访谈（8-10 位新生 + 5 位社团负责人） | 了解真实使用体验、挖掘迭代方向 |

## 竞品参考
